## ⚡ 快速标注流程

### 1. 初读划分
- 按自然段或场景划分
- 标记转折点

### 2. 依次标注
1. **叙事功能**（基础）
2. **叙事结构**（钩子、伏笔）
3. **角色关系**（人物相关）
4. **优先级**（P0/P1/P2）
5. **时空**（防迷失）

### 3. 三问法判断优先级
1. 删了主线还能懂吗？ → 否=P0, 是=P1/P2
2. 这是首次信息吗？ → 首次必留
3. 这段有多个功能吗？ → 都标上

## ⚠️ 常见错误

### ❌ 错误做法
1. **删除所有重复**：重复强调是手法，不是冗余
2. **忽略时空**：读者会迷失
3. **只标P0**：缺乏层次
4. **标签过细**：失去实用性

### ✅ 正确做法
1. **重复保留1-2次**：在关键时刻强调
2. **每段都标时空**：建立连续性
3. **三级优先级都有**：便于灵活浓缩
4. **标签适度**：够用即可

## 🎯 典型示例

### 示例1：铁律建立
```markdown
{
    在这个车队的人，必须谨记两条铁律！
    第一条：不要掉队！不要掉队！不要掉队！！！
    第二条：尽可能多的储备物资！！！
}

[核心故事设定（首次）] [伏笔] [重复强调x3]
[P0-骨架：铁律1] [P1-血肉：铁律2]
[地点：路上] [时间：白天]

浓缩：车队第一铁律-不要掉队，否则必死。
```

### 示例2：系统觉醒
```markdown
{
    一道信息出现在陈野的脑海之中。
    【升级毛毯需要消耗杀戮值129，是否需要升级？】
    ...自己觉醒的并不是序列能力，而是升级系统。
}

[故事推进-重大转折] [核心故事设定（首次）]
[钩子-悬念制造：有能力但没点数] [钩子-悬念释放：借贷]
[P0-骨架] [首次信息]
[地点：营地] [时间：夜晚]

浓缩：主角觉醒升级系统，需要杀戮点，可借贷300点。
```

### 示例3：对立建立
```markdown
{
    "野哥，你能不能让我坐你车斗里啊..."
    "不行！"陈野想都没想就拒绝了。
}

[故事推进] [人物登场：佳佳] [对立关系：陈野 vs 佳佳]
[人物塑造：主角果断] [伏笔]
[P0-骨架：冲突开始] [P2-皮肤：具体对话]
[地点：营地] [时间：清晨]

浓缩：瑜伽裤女想坐车被拒，矛盾开始。
```

## 💡 记住这些原则

1. **每段都有功能**：无功能=理解有误
2. **重复≠冗余**：可能是强调手法
3. **P0能构成主线**：单看P0能懂故事
4. **首次必留**：新信息优先级最高
5. **时空要追踪**：防止读者迷失
